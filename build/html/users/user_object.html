The User object¶

Each user belong to particular Application. User can have different bio, title, roles etc in different applications he belong to.

Field Description
_id The id of the User object
appId Id of application to which user belong
accountId Id of User Account.
picture Url to the user picture
fullName full name of the user
email Email address of the user
bio Bio of user
userName Internal user name, used for mentions.
settings Notifications settings
statistic User statistic according to his posts
title User title
roles Array of user roles.
createdOn Date when user was created (signed up)

User roles¶

  1. admin - can send invites, update billing information
  2. decisionMaker - can make decisions and see dashboard
